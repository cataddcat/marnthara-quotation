## 🚀 เริ่ม session ใหม่ — อ่านหน้านี้ก่อนเริ่มทุกครั้ง

**กุญแจ:** session ใหม่ "รู้บริบทเอง" ตั้งแต่ข้อความแรก — **ห้าม paste context/มาตรฐานยาว ๆ ซ้ำ** (เปลือง token ฟรี)

โหลดให้อัตโนมัติทุกครั้ง (คุณไม่ต้องทำ): `CLAUDE.md` (โปรเจกต์ + กฎ) · `MEMORY.md` (ตัวตน + workflow คุณ) ·
`.claude/commands/*` (`/probe-fix` · `/doc-name`) · `.claude/agents/*` (`design-reviewer`)
→ AI รู้อยู่แล้วว่าคุณเป็นเจ้าของร้าน · กฎดีไซน์ = colorful + contrast · คุณรัน CLI เอง — **ไม่ต้องบอกซ้ำ**

### พิมพ์ข้อความแรกแค่นี้ (เลือกตามงาน)
| งาน | ข้อความแรก |
|---|---|
| UI 1 จุด | `/probe-fix` + วาง Probe block (จากแอป กด **Alt+L**) |
| ตรวจ UI หลายไฟล์ | `ให้ design-reviewer ตรวจ <path>` |
| ชื่อไฟล์/เอกสาร export | `/doc-name <จุด>` |
| ฟีเจอร์/บั๊กทั่วไป | `<file:line> — <ปัญหา> · แค่จุดนี้ ไม่ต้องแตะที่อื่น` |
| สถาปัตยกรรม (save/pricing/หลายไฟล์) | เติม `อ่าน HANDOFF §1 ก่อน แล้ว…` |

### เช็ค 3 ข้อก่อนพิมพ์ (กันหลอน/หลุด/เปลือง)
1. **"เรื่องเดียว" ไหม?** — ไม่ใช่ → แยก session (`/clear` คั่น)
2. **ชี้จุดได้ไหม?** — `file:line` (Probe) หรือ **แปะรูป** > บรรยายด้วยคำ
3. **มีคำสั่งเฉพาะไหม?** — ดู [registry](#-คำสั่งผู้ช่วยทั้งหมดที่มี-registry--อัปเดตที่นี่เมื่อเพิ่มใหม่) ด้านล่าง → มี = ใช้คำสั่ง (มาตรฐานฝังแล้ว)

### อย่าทำ (ตัวเปลือง token)
❌ paste context/มาตรฐานยาว ๆ ซ้ำ · ❌ ยัดหลายงานไม่เกี่ยวกันในห้องเดียว · ❌ "ปรับทั้งแอปให้สวยขึ้น" (กว้างเกิน → AI เดา → หลอน)

> **สูตรเปิดห้องที่คุ้มสุด: ห้องใหม่ → "หนึ่งงาน + ชี้จุด (หรือคำสั่ง)" → จบ**
> ไม่ต้องเล่าว่าโปรเจกต์คืออะไร ไม่ต้องบอกมาตรฐาน — มันรู้แล้ว

## 🎯 TL;DR — จำแค่ภาพนี้ภาพเดียว

```
        กวาด                 เลือก                ยิง
   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
   │ design-      │    │   คุณ triage  │    │  /probe-fix  │
   │ reviewer     │ →  │  "1,3 ตกลง   │ →  │  แก้จุดเดียว  │
   │ (ตรวจหลายไฟล์)│    │   2 ไม่ทำ"    │    │  ให้แม่นเป๊ะ  │
   └──────────────┘    └──────────────┘    └──────────────┘
    หาว่าเสียตรงไหน       คุณคุมการตัดสินใจ      แก้ตรงเป๊ะ ถูก token
```

**กฎทอง 3 ข้อ:**
1. **พูดครั้งเดียว ใช้ตลอด** — มาตรฐานที่พูดซ้ำ (colorful/contrast) ฝังไว้ในเครื่องมือแล้ว ไม่ต้องพิมพ์ซ้ำ
2. **แยกห้องตามงาน** — งานคนละเรื่อง = session คนละห้อง (`/clear` คั่น) → context ไม่บวม
3. **ชี้เป๊ะ ไม่บรรยาย** — ใช้ Design Probe ชี้ `file:line` หรือ **แปะรูป** แทนการเล่าด้วยคำ

## 💸 ทำไม token หมดเร็ว (เข้าใจสาเหตุก่อน)

ทุกครั้งที่ AI ตอบ มันต้อง "อ่านซ้ำ" ของพวกนี้ **ทั้งหมด ทุกครั้ง**:

| สิ่งที่ถูกอ่านซ้ำทุก turn | ทำให้แพงเพราะ |
|---|---|
| สรุป session เก่า (ยิ่ง session ยาว ยิ่งใหญ่) | session มาราธอน = ทุก turn หลังจากนั้นแพงขึ้นเรื่อย ๆ |
| ไฟล์ที่เปิดค้างไว้ (อ่าน 10 ไฟล์ตอน "สแกน") | ผลสแกนลอยกินที่ตลอดทั้งห้อง |
| มาตรฐานที่คุณพิมพ์ซ้ำทุกครั้ง | "ใช้ DESIGN.md + colorful + contrast" × ทุกคำสั่ง |

**ทางแก้ = 3 กฎทองด้านบน** เครื่องมือที่เราสร้างคือตัวบังคับให้ทำตามกฎโดยอัตโนมัติ

## 🔁 COMBO ครบจบ — 3 ขั้น (หัวใจของทั้งหมด)

### สถานการณ์ตัวอย่าง: "การ์ดในหน้า RoomCard ดูจาง ๆ ปุ่มไม่ชัด อยากให้เข้ามาตรฐาน"

#### ขั้น 1 — กวาด 🧹 (ให้ design-reviewer ตรวจ)
พิมพ์ในห้องแชตปกติ:
```
ให้ design-reviewer ตรวจ src/components/workspace/RoomCard.tsx
```
มันจะอ่านไฟล์ **ในห้องของมันเอง** (ไม่กินที่ห้องหลัก) แล้วส่งกลับมาเป็น **รายการเรียงลำดับ**:
```
🔴 1. RoomCard.tsx:142 — ราคาใช้ text-muted-foreground (จาง) → ควร text-emerald-700 (เงิน=เขียว)
🔴 2. RoomCard.tsx:88  — การ์ดไม่มี border-border → เพิ่ม border ให้แยกจากพื้น
🟡 3. RoomCard.tsx:201 — ชิปจำนวนใช้ text-[11px] (<12px ผิดกฎ) → text-xs
🟢 4. RoomCard.tsx:55  — ระยะ gap แน่นไปนิดบนมือถือ
```

#### ขั้น 2 — เลือก ✅ (คุณ triage — ใช้รูปแบบที่คุณถนัด)
```
1, 2, 3 ตกลง   4 ไม่ต้อง
```
> คุณคือคนตัดสินใจ design เสมอ AI ไม่เดาเอง

#### ขั้น 3 — ยิง 🎯 (แก้ตามที่อนุมัติ)
AI แก้เฉพาะ 3 ข้อที่อนุมัติ แล้วบอกท้ายว่า **ให้คุณรัน** `npm run lint && npm run test:run && npm run build`
(AI ไม่รัน CLI เอง — คุณรันเอง ตาม workflow ประจำ)

**ถ้าอยากแก้จุดเดียวแบบเจาะลึก** (ไม่ผ่าน reviewer) → ใช้ `/probe-fix` แทน (ดูหัวข้อถัดไป)

## 🛠️ เครื่องมือ 2 ชิ้นที่เราสร้าง

### 1️⃣ `/probe-fix` — ยิงเป้าจุดเดียว ให้แม่นเป๊ะ

| หัวข้อ | รายละเอียด |
|---|---|
| **คืออะไร** | คำสั่งลัด (slash command) ที่ฝังมาตรฐาน DESIGN.md (colorful + contrast + ห้ามไอคอนมั่ว) ไว้แล้ว |
| **อยู่ที่ไหน** | `.claude/commands/probe-fix.md` (อยู่ใน repo แล้ว — commit `af57d56`) |
| **ใช้เมื่อไหร่** | เห็นจุดที่ไม่โอเคแล้ว 1 จุด อยากแก้ให้ตรงมาตรฐานแบบแม่น ๆ |
| **ใช้ยังไง** | 1) เปิดแอป `npm run dev` 2) กด **Alt+L** (หรือปุ่ม "Probe" ลอย) 3) คลิก element ที่เพี้ยน → มันก็อปบล็อกให้ 4) ในแชตพิมพ์ `/probe-fix` แล้ว **วางบล็อกที่ก็อปมา** |

**ตัวอย่างจริง** — พิมพ์ในแชต:
```
/probe-fix
src/components/ui/Modal.tsx:250
"ผ้าม่านมู่ลี่ไม้ม่านม้วน..." · 16px / lh 24px / w400
classes: (none) — inherited
role: Title / Body (ok)
--- ไอคอนมันไม่ตรงสินค้าเลย เอาออก ทำเป็น grid 2 คอลัมน์ ตัวอักษรล้วน
```
AI จะอ่าน **เฉพาะรอบบรรทัด 250** (ไม่อ่านทั้งไฟล์), แก้ตามมาตรฐานที่ฝังไว้, ไม่ต้องให้คุณพิมพ์ "ใช้ DESIGN.md
+ colorful + contrast" ซ้ำอีก

---

### 2️⃣ `design-reviewer` — กวาดหาทั้งหน้า/หลายไฟล์

| หัวข้อ | รายละเอียด |
|---|---|
| **คืออะไร** | ผู้ช่วยตรวจ (subagent) แบบ **อ่านอย่างเดียว ไม่แก้ไฟล์** — ตรวจ UI เทียบ DESIGN.md แล้วคืนรายการ |
| **อยู่ที่ไหน** | `.claude/agents/design-reviewer.md` (อยู่ใน repo แล้ว — commit `af57d56`) |
| **ใช้เมื่อไหร่** | ยังไม่รู้ว่ามีปัญหาตรงไหนบ้าง อยากให้กวาดตรวจหลายไฟล์/ทั้งหน้าทีเดียว |
| **ใช้ยังไง** | พิมพ์ในแชตปกติ เอ่ยชื่อมัน: `"ให้ design-reviewer ตรวจ <ไฟล์/โฟลเดอร์>"` |

**ตัวอย่างจริง** — พิมพ์ในแชต:
```
ให้ design-reviewer ตรวจทุกไฟล์ใน src/components/workspace/
```
มันจะกวาดทุกไฟล์ **ในห้องแยกของมันเอง** (ห้องหลักคุณไม่บวม) แล้วคืน numbered list 🔴/🟡/🟢 →
คุณ triage → ห้องหลักค่อยแก้ตามที่อนุมัติ

> **ทำไมต้องแยก subagent?** ถ้าให้ห้องหลักอ่าน 20 ไฟล์เอง ไฟล์ทั้ง 20 จะค้างกินที่ทุก turn ถัดไป
> แต่ subagent อ่านในห้องมัน เสร็จแล้วส่งกลับมาแค่ "ลิสต์สรุป" — ห้องหลักสะอาด

## 🧠 ความต่างของ 2 เครื่องมือ (อย่าสับสน)

| | `design-reviewer` | `/probe-fix` |
|---|---|---|
| **บทบาท** | กวาดหา (breadth) | ยิงเป้า (precision) |
| **ขอบเขต** | หลายไฟล์/ทั้งหน้า | จุดเดียวที่ Probe ชี้ |
| **แก้ไฟล์ไหม** | ❌ ไม่แก้ (รายงานอย่างเดียว) | ✅ แก้ให้ |
| **ใช้ตอน** | ยังไม่รู้ว่าเสียตรงไหน | รู้แล้วว่าจุดไหนเสีย |
| **เรียกยังไง** | พิมพ์ "ให้ design-reviewer ตรวจ…" | พิมพ์ `/probe-fix` + วาง Probe |

จำง่าย ๆ: **reviewer หาเจอ → probe-fix ยิงเก็บ**

## 📂 คำสั่ง/ผู้ช่วยทั้งหมดที่มี (registry — อัปเดตที่นี่เมื่อเพิ่มใหม่)

| ใช้ | ชนิด | ทำอะไร | ไฟล์ |
|---|---|---|---|
| `/probe-fix` | command | แก้ UI 1 จุดตาม Design Probe ให้เข้ามาตรฐาน DESIGN.md | `.claude/commands/probe-fix.md` |
| `/doc-name` | command | ตั้งชื่อไฟล์/เอกสาร export แบบสืบย้อนได้ (ลูกค้า + รหัสไม่ซ้ำ + sanitize) | `.claude/commands/doc-name.md` |
| `design-reviewer` | subagent | กวาดตรวจ UI หลายไฟล์เทียบ DESIGN.md → คืนรายการจัดลำดับ | `.claude/agents/design-reviewer.md` |

> **เพิ่มคำสั่งใหม่เมื่อไหร่** → เพิ่มแถวที่นี่ + ลิงก์ใน "ไฟล์ที่เกี่ยวข้อง" ท้ายไฟล์ (ที่เดียว ไม่ต้องตามแก้หลายที่)
>
> หลักจำ: **มีคำสั่งเฉพาะ → ใช้คำสั่ง** (มาตรฐานฝังแล้ว ถูก token สุด) · **ไม่มี → เขียน prompt ที่ดี**

## 🪙 5 นิสัยประหยัด token (ทำได้ทันที)

| # | ทำแบบนี้ ✅ | อย่าทำแบบนี้ ❌ | ประหยัดเพราะ |
|---|---|---|---|
| 1 | **แปะรูป** หน้าจอที่เพี้ยน | เล่าด้วยคำ "การ์ดดูแบน ๆ" | AI เห็นเอง สั้นกว่า แม่นกว่า |
| 2 | **ชี้ `file:line`** ด้วย Probe | "แก้ Modal ให้สวยขึ้น" | ไม่ต้องไล่ค้นทั้งโปรเจกต์ |
| 3 | งานใหม่ = **`/clear` ขึ้นห้องใหม่** | ต่อทุกงานในห้องเดียวยาว ๆ | สรุป session ไม่บวม |
| 4 | "สแกน/หา" = **โยนให้ subagent** | ให้ห้องหลักอ่านเองสิบไฟล์ | ไฟล์ไม่ค้างกินที่ห้องหลัก |
| 5 | งานเล็ก (ค้นหา/อธิบาย) = **`/model haiku`** | ใช้ Opus ทุกงาน | Haiku ถูกกว่า ~5 เท่า |

**โบนัส — `#` เพิ่มความจำถาวร:** เจอกฎใหม่ พิมพ์ขึ้นต้นด้วย `#` เช่น
`# ปุ่มเลือกประเภทใช้ text grid 2 คอลัมน์เสมอ ห้ามไอคอนมั่ว` → AI จำตลอดไป ไม่ต้องสอนซ้ำ

## 📋 เทมเพลตคัดลอกใช้ได้เลย

**ตรวจทั้งหน้า:**
```
ให้ design-reviewer ตรวจ src/components/workspace/RoomCard.tsx แล้วสรุปเป็นรายการ
```

**แก้จุดเดียว (หลังก็อปจาก Probe):**
```
/probe-fix
<วางบล็อกจาก Design Probe ที่นี่>
--- <สั่งเพิ่มถ้าต้องการ เช่น "เอาไอคอนออก ทำเป็นตัวอักษรล้วน">
```

**สั่งงานที่ดี (ชี้เป๊ะ + ขอบเขตชัด):**
```
RoomCard.tsx:142 — ราคาจางไป ขอ text-emerald-700 (เงิน=เขียว)
RoomCard.tsx:88  — การ์ดไม่มีขอบ ขอ border-border
แค่ 2 จุดนี้ ไม่ต้องแตะที่อื่น
```

**ปิดท้ายทุกงาน (คุณรัน gate เอง):**
```
แก้เสร็จบอกผมว่าให้รันอะไร ผมรัน lint/test/build + commit เอง
```
