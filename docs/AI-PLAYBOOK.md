## 1. เอกสารนี้คืออะไร / ใครควรอ่าน

ถ้าคุณ **ไม่ใช่โปรแกรมเมอร์อาชีพ** แต่ใช้ AI agent (เช่น Claude Code) สร้างหรือดูแลแอป/เว็บของตัวเอง —
คู่มือนี้เขียนเพื่อคุณ มันสรุปวิธี **สั่งงาน AI ให้ได้งานดี เข้าใจบริบท ไม่หลอน และไม่เปลือง token**
โดยไม่ต้องรู้ศัพท์เทคนิคลึก ๆ

**แก่นของทั้งเล่มมีประโยคเดียว:**

> 🎯 **พูดให้น้อย ชี้ให้แม่น และเก็บสิ่งที่พูดซ้ำให้กลายเป็นเครื่องมือ**

## 2. ทำไมต้องสนใจ — token economics ฉบับเข้าใจง่าย

AI ไม่ได้ "จำ" บทสนทนาแบบคน — **ทุกครั้งที่มันตอบ มันต้องอ่านทุกอย่างในห้องแชตใหม่ทั้งหมด** (เรียกว่า context)
ยิ่งห้องมีของเยอะ แต่ละคำตอบยิ่งแพง

3 ตัวกินงบเงียบ ๆ:

| ตัวกิน | เกิดเพราะ |
|---|---|
| **Session ยาว** | ห้องสะสมประวัติเยอะ → ทุกคำตอบหลังจากนั้นแพงขึ้นเรื่อย ๆ |
| **คำสั่งกว้าง** | "ปรับทั้งแอปให้สวยขึ้น" → AI ต้องเดา + อ่านไฟล์มหาศาล + เสี่ยงหลอน |
| **พูดมาตรฐานซ้ำ** | บอก "ทำให้สวย ใช้สีแบบนี้ อย่าลืม..." ทุกครั้ง = จ่ายซ้ำทุกครั้ง |

> 🔸**ตัวอย่าง:** เคยสั่งงานเดียวแต่ใช้โควต้า 5 ชั่วโมงไป **46%** สาเหตุคือ session ยาวมาราธอน +
> เปิดหลายไฟล์ค้างไว้ + พิมพ์มาตรฐานเดิมซ้ำทุกคำสั่ง

**ข่าวดี:** ทั้ง 3 ตัวแก้ได้ด้วยนิสัย + เครื่องมือในข้อ 4–5

## 3. หลักคิดแกน: ระบบจำบริบทให้ — อย่า brief ซ้ำ

AI agent สมัยใหม่มี **2 ชั้นความจำ** เข้าใจสองชั้นนี้ แล้วคุณจะเลิกพิมพ์อะไรซ้ำ ๆ:

| ชั้น | คืออะไร | โหลดเมื่อไหร่ |
|---|---|---|
| **① อ่านอัตโนมัติทุก session** | ไฟล์คำสั่งโปรเจกต์ (เช่น `CLAUDE.md`) + ความจำถาวร (memory) | ทุกครั้ง โดยไม่ต้องสั่ง |
| **② อ่านเมื่อจำเป็น** | เอกสาร/ไฟล์อื่น ๆ ที่ชั้น ① ชี้ทางไว้ | ตอนงานต้องใช้เท่านั้น (กันเปลือง) |

แปลว่า: ถ้าคุณ **เขียนบริบทสำคัญลงชั้น ① ครั้งเดียว** (ใครคุณคือใคร · โปรเจกต์ทำอะไร · กฎที่ต้องตามเสมอ)
AI จะรู้มันทุก session โดยที่คุณ **ไม่ต้องเล่าซ้ำ**

> 🧠 **พูดครั้งเดียว ใช้ตลอด** — นี่คือหัวใจของการประหยัด

## 4. 5 นิสัยพกพาได้ (ทำได้ทันที ทุกโปรเจกต์)

| # | นิสัย | ทำไม |
|---|---|---|
| 1 | **งานเดียวต่อ 1 session** — งานไม่เกี่ยวกันให้ขึ้นห้องใหม่ (`/clear`) | ห้องไม่สะสมประวัติ → ทุกคำตอบถูกลง |
| 2 | **ชี้เป๊ะ ไม่บรรยาย** — บอกตำแหน่งไฟล์/บรรทัด หรือ **แปะรูป** | AI ไม่ต้องไล่ค้น + เห็นปัญหาตรง ๆ |
| 3 | **งานค้นหา/กวาด → ใช้ผู้ช่วยแยก (sub-agent)** | ไฟล์ที่อ่านไม่ค้างกินที่ห้องหลัก |
| 4 | **เลือกโมเดลตามงาน** — งานค้นหา/อธิบายใช้รุ่นถูก · งานตัดสินใจใช้รุ่นเก่ง | จ่ายแพงเฉพาะตอนที่ต้องคิดหนัก |
| 5 | **บันทึก decision ลงความจำถาวร (memory)** | ข้ามรอบได้ ไม่ต้องสอนซ้ำ session หน้า |

**ขยายนิสัยข้อ 2 (สำคัญสุด):**
- ❌ "ปุ่มมันดูแปลก ๆ ช่วยแก้หน่อย"
- ✅ "ปุ่มในไฟล์ X บรรทัด 142 สีจางไป ขอให้เข้มขึ้น — แค่จุดนี้"
- ✅ หรือ **แปะ screenshot** ของจุดที่เพี้ยน (สำหรับงานหน้าตา รูปภาพชนะคำบรรยายเสมอ)

## 5. แพทเทิร์นหัวใจ: เปลี่ยน "สิ่งที่พูดซ้ำ" → "สินทรัพย์ที่ใช้ซ้ำ"

นี่คือไอเดียที่ทรงพลังที่สุดในเล่ม **ทุกอย่างที่คุณพิมพ์ซ้ำบ่อย ๆ ให้ย้ายไปเก็บเป็นเครื่องมือ** แล้วเรียกใช้สั้น ๆ

| สิ่งที่พูดซ้ำ | ย้ายไปเก็บเป็น | ได้อะไร |
|---|---|---|
| "โปรเจกต์นี้คือ... กฎคือ..." | **ไฟล์คำสั่งโปรเจกต์** (เช่น `CLAUDE.md`) | AI รู้บริบททุก session อัตโนมัติ |
| "ผมชอบแบบนี้ / ตัดสินใจแบบนี้แล้ว" | **ความจำถาวร (memory)** | preference อยู่ข้ามรอบ |
| ขั้นตอนงานที่ทำซ้ำ + มาตรฐานของมัน | **คำสั่งลัดของตัวเอง (custom slash command)** | พิมพ์คำเดียว มาตรฐานฝังในตัว |
| บทบาท "ตรวจ/รีวิว" ที่ทำซ้ำ | **ผู้ช่วยเฉพาะทาง (custom sub-agent)** | กวาดงานในห้องแยก ห้องหลักไม่บวม |

> 🔸**ตัวอย่าง (Marnthara):** แทนที่จะพิมพ์ "แก้ให้เข้ามาตรฐานดีไซน์ สีสด คอนทราสต์สูง ไม่ใช้ไอคอนมั่ว"
> ทุกครั้ง → เก็บเป็นคำสั่งลัด `/probe-fix` (มาตรฐานฝังในแล้ว) และมีผู้ช่วย `design-reviewer`
> ไว้กวาดตรวจหน้าจอ. รายละเอียดจริงดูที่ [AI-WORKFLOW.md](AI-WORKFLOW.md)

**กฎเลือก:** งานทำ **ครั้งเดียว** → เขียนเป็นคำสั่งที่ดี (ข้อ 4) · งานทำ **ซ้ำบ่อย** → เก็บเป็นคำสั่งลัด/ผู้ช่วย

## 6. คอมโบครบจบ: กวาด → เลือก → ยิง

ลูปนี้ใช้ได้กับงานปรับปรุงเกือบทุกชนิด (ไม่ใช่แค่ดีไซน์):

```
     กวาด                  เลือก                  ยิง
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ ผู้ช่วยตรวจ    │     │  คุณตัดสินใจ   │     │  แก้จุดที่    │
│ (sub-agent)  │ →   │ "ข้อ 1,3 เอา │ →   │  อนุมัติ ตรง  │
│ คืนรายการ     │     │  ข้อ 2 ไม่เอา" │     │  ๆ เป๊ะ ๆ     │
└──────────────┘     └──────────────┘     └──────────────┘
 หาว่าเสียตรงไหน        คนคุมการตัดสินใจ        ถูก token เพราะชี้แล้ว
```

1. **กวาด** — ให้ผู้ช่วยแยกตรวจ แล้วคืน **รายการจัดลำดับ** (ในห้องของมัน ห้องหลักไม่บวม)
2. **เลือก** — คุณ triage สั้น ๆ ("เอา 1,3 · ไม่เอา 2") — **คนเป็นคนตัดสินใจเสมอ**
3. **ยิง** — แก้เฉพาะที่อนุมัติ ตรงจุด

## 7. เปิด session ใหม่ยังไงให้คุ้ม

เพราะระบบจำบริบทให้แล้ว (ข้อ 3) — **คุณไม่ต้อง brief อะไรยาว ๆ** บอกแค่ 3 อย่าง:

1. **อะไร** — งานเดียว
2. **ตรงไหน** — ไฟล์/บรรทัด หรือ รูป
3. **ขอบเขต** — "แค่จุดนี้ ไม่ต้องแตะที่อื่น"

**ตัวอย่างข้อความแรกที่สมบูรณ์:**
```
ไฟล์ X บรรทัด 142 — ราคาจางไป ขอให้เข้มขึ้น · แค่จุดนี้
```

**ไม่ต้องบอก** (ระบบรู้แล้ว): โปรเจกต์คืออะไร · มาตรฐาน/สไตล์ · workflow ส่วนตัวของคุณ

## 8. อย่าทำ (anti-patterns เปลือง token)

- ❌ **paste บริบท/มาตรฐานยาว ๆ ซ้ำทุกครั้ง** — เก็บลงไฟล์คำสั่งโปรเจกต์แทน (ข้อ 5)
- ❌ **ยัดหลายงานไม่เกี่ยวกันในห้องเดียว** — แยก session
- ❌ **สั่งกว้าง** เช่น "ทำให้ดีขึ้นทั้งหมด" — AI เดา → หลอน → diff บานปลาย
- ❌ **ปล่อย session ยาวข้ามวัน** — งานจบแล้วขึ้นห้องใหม่
- ❌ **ให้ห้องหลักไล่อ่าน 10–20 ไฟล์เพื่อ "สแกนหา"** — โยนให้ผู้ช่วยแยก

## 9. Starter kit — ตั้งค่าโปรเจกต์ใหม่ให้พร้อมใน 4 ขั้น

ทำครั้งเดียวตอนเริ่มโปรเจกต์ แล้วเก็บเกี่ยวความประหยัดไปตลอด:

1. **สร้างไฟล์คำสั่งโปรเจกต์** (เช่น `CLAUDE.md` ที่ราก repo) — เขียน: โปรเจกต์ทำอะไร · โครงสร้างหลัก ·
   กฎที่ต้องตามเสมอ · คำสั่งทดสอบ/บิลด์ → AI อ่านอัตโนมัติทุก session
2. **เปิดความจำถาวร (memory)** — ให้ AI จดตัวตนคุณ + preference + decision สำคัญ ที่ไม่ได้อยู่ในโค้ด
3. **ทำคำสั่งลัด 1–2 อัน** สำหรับ "งานที่คุณพิมพ์ซ้ำบ่อยที่สุด" — ฝังมาตรฐานของงานนั้นไว้ในคำสั่ง
4. **ทำผู้ช่วยตรวจ (sub-agent)** ถ้ามีงานรีวิว/ตรวจซ้ำ ๆ (เช่น ตรวจหน้าตา ตรวจความปลอดภัย) — ให้มันกวาดในห้องแยก

> 🔸**ตัวอย่าง (Marnthara):** มี `CLAUDE.md` (บริบท) + memory (preference) + คำสั่งลัด `/probe-fix`·`/doc-name`
> + ผู้ช่วย `design-reviewer` — ครบทั้ง 4 ขั้น ดูของจริงที่ [AI-WORKFLOW.md](AI-WORKFLOW.md)

## 10. Cheat-sheet พกพา — สรุปบรรทัดเดียว

| อยากได้ผลแบบ… | ทำ |
|---|---|
| AI เข้าใจบริบทโดยไม่ต้องเล่า | เขียนไฟล์คำสั่งโปรเจกต์ครั้งเดียว |
| ไม่หลอน ไม่หลุด | ชี้ไฟล์/บรรทัด หรือ แปะรูป · 1 งาน/1 session |
| ไม่เปลือง token | session สั้น · งานกวาดโยน sub-agent · เลือกโมเดลตามงาน |
| ไม่พิมพ์มาตรฐานซ้ำ | เก็บเป็นคำสั่งลัด/ผู้ช่วย |
| ปรับปรุงเป็นชุด | คอมโบ กวาด → เลือก → ยิง |
| เปิดห้องใหม่ | บอกแค่ "งานนี้ ตรงนี้ แค่นี้" |
