# 🧪 QA Checklist — Marnthara Smart Quotation (+ Catalog Tool)

เช็กลิสต์ทดสอบ manual "ทุกซอกทุกมุม" — ไล่ทีละข้อ ติ๊ก ✅/❌ ในคอลัมน์ **📱 มือถือ (390px)** และ
**🖥️ เดสก์ท็อป**. ใช้คู่กับเทสต์อัตโนมัติ (lint · unit 697 · E2E 15 · build) ที่เป็น baseline แล้ว.

> **วิธีตั้งจอมือถือ:** `npm run dev` → ปุ่ม **"จอมือถือ"** (DevInspector มุมล่างซ้าย) เปิด popup 390×844 +
> ปุ่ม **"Notch"** จำลอง safe-area. ตรวจสี/คอนทราสต์ด้วย **Design Probe** (Alt+L).

## 9) Catalog Tool (`marnthara-catalog-tool`, port 3100)
> ⚠️ เขียน Firestore จริง — ทดสอบด้วยแถว **`ZZ-TEST-*`** แล้วลบทิ้ง (อย่าแตะข้อมูลจริง)

| ข้อทดสอบ | คาดหวัง | ✅ |
|---|---|:--:|
| `/` อัปรูป/วางข้อความใบราคา → AI สังเคราะห์ | ได้แถว review (เลือก provider/รุ่นได้) | ☐ |
| review → ตรวจเทียบคลัง (diff) → อนุมัติเขียน | เขียนเฉพาะที่ติ๊ก, stamp captured_at | ☐ |
| `/catalog` CRUD + bulk + lock + ค้นหา + กรองหมวด | แก้/ลบ/เพิ่ม batch ได้, lock กันพลาด | ☐ |
| 🔴 `/customers` โหลดรายการ | เรียงชื่อ, โชว์ UUID อ่านอย่างเดียว | ☐ |
| 🔴 `/customers` ปลดล็อก → เพิ่ม/แก้/ลบ (ZZ-TEST) | บันทึกได้, UUID สร้างตอนบันทึกครั้งแรก | ☐ |
| 🔴 `/customers` rename รหัส | ลบ doc เดิม + สร้างใหม่ | ☐ |
| Basic Auth (ถ้าตั้ง env บน Vercel) | ไม่ใส่รหัส = 401 ทุกหน้า/ทุก API | ☐ |
| 🔴 loop: แก้ที่ tool → แอปหลักเห็น realtime | sign-in บัญชีเดียวกัน เห็นข้อมูลตรง | ☐ |
| ลบแถว ZZ-TEST ทั้งหมดหลังเทสต์ | catalog/customers สะอาด | ☐ |
