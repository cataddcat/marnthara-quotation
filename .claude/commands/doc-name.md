ทำให้ชื่อไฟล์/เอกสารที่ผมชี้ เข้ามาตรฐานการตั้งชื่อแบบ "สืบย้อนหลังได้และไม่ซ้ำ"

## จุดที่ให้แก้
$ARGUMENTS

## ก่อนแตะ
อ่าน HANDOFF §1 (save/backup flow) — งานนี้แตะ data persistence/export

## มาตรฐานชื่อเอกสาร (บังคับ)
ทุกชื่อไฟล์/เอกสารที่ export ต้องมี 4 ส่วน เรียงตามนี้:
`<ประเภท>_<ชื่อลูกค้า>_<รหัสเอกสาร>_<YYYYMMDD>`
ตัวอย่าง: `mtr-backup_สมชาย_C0007-001_20260609.json`

- **ชื่อลูกค้า**: ดึงจาก Customer (store) → sanitize ให้เป็น filename ปลอดภัย
  (แทน `/ \ : * ? " < > |` · เว้นวรรค · emoji → `-`)
- **รหัสเอกสาร**: ต้อง **ไม่ซ้ำ** และสืบกลับหาลูกค้าได้เสมอ แม้ลูกค้าชื่อเดิม/คนเดิม
- ไม่มีชื่อลูกค้า → fallback อ่านออก เช่น `ไม่ระบุชื่อ` (รหัสยังต้องไม่ซ้ำ)
- backup กับ ใบเสนอราคา ใช้สคีมารหัสเดียวกัน (สืบโยงกันได้)

## ต้องเสนอให้ผมเลือกก่อนลงมือ (อย่าเดา)
1. Customer มี stable `id` หรือยัง (เช็ค `src/types.ts`) — ถ้าไม่มี จะสร้าง + เก็บที่ไหน
2. รูปแบบรหัส: running ต่อลูกค้า (`C0007-001`) / `ชื่อ-วันที่-ลำดับ` / อื่น ๆ
3. ตรวจซ้ำ = **เตือน** / **auto +1** / **บล็อก** — เลือกพฤติกรรมเดียว

## ห้าม
- แตะ pricing / save logic
- เปลี่ยนโครงสร้าง backup JSON จนของเก่า restore ไม่ได้

## วิธีทำ
1. อ่านเฉพาะรอบบรรทัดที่ชี้ (blast radius เล็กที่สุด)
2. เสนอ 3 ข้อตัดสินใจด้านบนก่อน → รอผมเลือก → ค่อยแก้จริง
3. อย่ารัน CLI — ผมรัน `npm run lint && npm run test:run && npm run build` เอง
4. สรุปท้าย: เดิม→ใหม่ · ตัวอย่างชื่อไฟล์จริง · ทวนยังไง (export ลูกค้าชื่อเดียวกัน 2 ครั้ง → รหัสต้องต่างกัน) · ให้ผมรันอะไร
